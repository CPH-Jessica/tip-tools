Layer 3: The Project

A project is the workspace where everything comes together for a specific book or series. It's the most specific layer — narrowed down to one creative job.

  • Instructions — book premise, character list, POV rules, specific constraints set once
  • Knowledge files — outline, character profiles, previous chapters, voice skill uploaded
  • Conversations — drafting, brainstorming, editing sessions ongoing

Key insight: Your skills from Week 2 get uploaded here as knowledge files. The project doesn't replace your skills — it houses them alongside book-specific context.

Layer 2: Skills

Skills are the how-to documents you built in Week 2. They tell Claude how to do specific jobs — write in your voice, give beta feedback, edit with your standards.

  • Voice skill — how this pen name sounds from Week 2
  • Beta reader skill — how to evaluate your writing from Week 2
  • Editing skills — developmental, line edit, dialogue polish future

Key insight: One voice skill can be used in multiple projects. Your Sarah Westbrook voice skill goes into every Sarah Westbrook book project.

Layer 1: Preferences

Preferences are global — they follow you into every project and every conversation. Claude always knows who you are and how you like to work.

  • Communication style — direct, no fluff, push back on weak ideas from Week 1
  • Work habits — how you process, make decisions, handle feedback from Week 1
  • Personal context — your background, goals, what matters to you from Week 1

Key insight: You don't need to repeat preferences in project instructions. They're already there. Projects add specificity on top of what Claude already knows about you.

Project Instructions

Instructions are the first thing Claude reads when you start a conversation in the project. They set the context, rules, and constraints for everything that happens inside.

For a book project, instructions include:

  • Pen name and genre
  • Book premise — 2-3 sentence elevator pitch
  • Main characters — names, roles, brief descriptions
  • POV, tense, and structural rules
  • What Claude should and shouldn't do in this project
  • Current status — what chapter you're on, what's been drafted

Pro tip: Instructions should be concise — under 1,000 words. If you're writing a novel's worth of instructions, some of that belongs in a knowledge file instead.

Knowledge Files

Knowledge files are documents Claude can reference during any conversation in the project. Upload them once, use them forever.

For a book project, knowledge files might include:

  • Your pen name voice skill (from Week 2)
  • Your beta reader skill (from Week 2)
  • Chapter-by-chapter outline
  • Character profiles and relationship map
  • World-building notes or setting details
  • Previously drafted chapters (for continuity)
  • Research notes

Pro tip: Claude reads knowledge files when they're relevant to the conversation. Label your files clearly so Claude knows which one to reference.

Conversations

Every conversation inside a project inherits the instructions and can access the knowledge files. You can run multiple conversations for different purposes.

Common conversation types in a book project:

  • Drafting session — writing chapters with Claude
  • Brainstorming session — plot problems, character development
  • Beta read session — feeding chapters to your beta reader skill
  • Editing session — line edits, dialogue polish, pacing review
  • Research session — setting details, historical accuracy

Pro tip: Start a new conversation for each major task rather than using one endless thread. Claude stays sharper with focused conversations.
